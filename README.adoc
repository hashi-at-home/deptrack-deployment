This repository contains the deployment definition of the dependency tracker service in the security plane of the platform.

== Architecture

This deploys the Dependency Track API server into the platform premise, and the UI on the edge.
It relies on an ingress router and a database in the data plane.

== Notes

. Requires KEK to be shared across all instances. Generate once and share. Added to `vault kv get -format=json hashiatho.me-v2/security_plane/deptrack/secret_management_kek`

. The key needs to be placed in the defined source directory by a Vault lookup, and allow the deployment to fail if not found.
